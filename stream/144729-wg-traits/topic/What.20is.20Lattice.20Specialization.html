bjorn3 (Jan 24 2022 at 18:13):

I think it is roughly the following:

trait Foo {
    fn foo(&self);
}

impl<T> Foo for T {
    default fn foo(&self) {}
}

trait Bar {}
trait Baz {}

// without lattice specialization either of the following implementations isn't allowed as they intersect:
impl<T: Bar> Foo for T {
    default fn foo(&self) {}
}


impl<T: Baz> Foo for T {
    default fn foo(&self) {}
}

// with lattice specialization both are allowed for as long as there is another implementation for the intersection like this:
impl<T: Bar + Baz> Foo for T {
    fn foo(&self) {}
}
