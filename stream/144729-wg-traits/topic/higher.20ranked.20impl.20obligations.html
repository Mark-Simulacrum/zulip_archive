Topic: higher ranked impl obligations

Boxy [she/her] (Sep 02 2021 at 17:38):

I came across some code (from #82219) :

pub trait Foo<'d> {}

impl<'c> Foo<'c> for () {}
impl<'f> Foo<'f> for u8 {}
impl<'e, T> Foo<'e> for Option<T> where T: Foo<'e> {}

pub fn bar<T>()
where
    T: for<'b> Foo<'b>,
{ }

fn baz<'a>()
where
    (): Foo<'a>,
{
    bar::<Option<()>>();
}

which errors because it cant satisfy (): for<'a> Foo<'a>.
It seems to happen because after selecting the impl to prove Binder(<Option<()> as Foo<'b>>, ['b]) we generate this obligation in impl_or_trait_obligations:
Binder(<() as Foo<'_>>, [])
which we then fulfill using the where clause (): Foo<'a>
It seems to me like we ought to be making an obligation of (): for<'a> Foo<'a> instead but im not very familiar with this code so am not sure both how to do that or if that's actually the solution/problem here.
