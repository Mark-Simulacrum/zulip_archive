oli (Oct 08 2021 at 15:02):

Hi and welcome!

I'm gonna start out with the bad news: we are really low on review capacity at present, so I would propose not to write new mir opts :wink:

That said, there's lots of things to do that have a noticeable impact on users and rustc devs. If you want to start with a refactoring, there is e.g the mir pass manager PR that needs a revival (https://github.com/rust-lang/rust/pull/77665). If you prefer building something new, there is the general idea of a peephole optimization framework (https://github.com/rust-lang/rust/pull/86525). If you'd rather start out with something small, there is https://github.com/rust-lang/rust/issues/85180 which should make writing opts easier.

Or if you have your own ideas, please don't hesitate to bring them up.
