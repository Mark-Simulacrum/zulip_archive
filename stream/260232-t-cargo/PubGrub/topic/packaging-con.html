What do you think of this proposal? is there another subject in particular we should focus on?

Title: Building a flexible dependency solver in Rust
Abstract: Dependency solving is a hard problem, especially when mixed with additional features such as optional dependencies, multiple versions or availability of pre-releases. We present a rewrite from scratch of a recent algorithm called PubGrub, as a Rust library aiming at great performances and flexibility for reuse. We will dive into its core mechanisms, its high-level usage, as well as the ideas enabling behavioral extensions such as optional dependencies, entirely in user space without changing the library API.
