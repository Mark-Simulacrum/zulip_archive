Didrik Nordstrom (Feb 25 2021 at 22:03):

My proposal for the async book in 2021 is a major restructure, using the Rust Book as precedent:

- The target audience is people that are familiar with, but not necessarily experts, on Rust
- Strike a balance between "skimmable reference" (look up when you need to) and "narrative" (read back-to-back)
- Describe the current state of Async Rust, warts and all (caveats and workarounds)
- No prior experience with asynchronous programming is necessary
- Concepts first, so that the reader gets intuition for how to build general purpose async apps and libs
- Code samples interspersed for demonstration purposes
- Highlight differences between async and sync code

It is not:

  • A guide for people writing custom executors.
  • A tutorial for building a specific type of application.
  • A replacement for individual runtime documentation.

Not included:

  • User journeys.

Perhaps we can add this to an "Restructuring the async book" section in the vision doc, so that it gets visibility and people can review it? WDYT?
