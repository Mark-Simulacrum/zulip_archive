Yoshua Wuyts (Apr 08 2021 at 14:07):

Somewhat related to https://rust-lang.zulipchat.com/#narrow/stream/187312-wg-async-foundations/topic/stories.20of.20portability I think we could use a story about using platform-native runtimes.

For example the Windows platform provides a wide array of facilities for writing native GUI applications that work asynchronously (e.g. [WinUI 3][1]). This relies on a different set of system calls and has different considerations than when writing server workloads.

I'm not sure how far stories have (or intend to dip into) abstractions such as "Task" and "runtime", but we probably should probably cover the need to be able to swap backing runtimes by end-users when used in different environments and workloads. This includes WinUI3, but also: Browser WASM, GNOME, precisely tuned servers, and more.

[1]: https://youtu.be/OSYXGyMw9GE
