Just FYI/Discussion...

I got a message from rustbot today:

Ping from triage:
@cehteh can you please address the comments and set S-waiting-on-review when you're done? Thank you.

This left me a bit puzzled about how to do this. I known that 'labels' are a github feature, but i didn't seen any UI for changing those. Searching for 'github set label' was not really helpful either. Thus I just asked on the PR how to handle this. Thankfully some kind soul (thanks hkratz) gave me the hint (and by that did this for me):

 Like this:
@rustbot label -S-waiting-on-author +S-waiting-on-review

(rustbot documentation)

Instead it would be nice when rustbot could give such an hint in the first place:

Ping from triage:
@cehteh can you please address the comments then reply with '@rustbot label -S-waiting-on-author +S-waiting-on-review'

Looks to me like some very simple change which would make contribution a lot simpler (and bind less human resources, petting noobs like me).

Furthermore, a bit more complicated: How about a '@rustbot done' command, which just scans what it requested most recently from the user and then applies that (Of course rustbots helpful hint should then include '... or reply with @rustbot done).
