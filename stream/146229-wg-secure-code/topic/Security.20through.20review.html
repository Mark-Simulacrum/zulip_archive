Hi all, I've wrote an xtask to formalize upgrading dependencies of a large project I'm involved in.

As it stands, the script simply runs upgrade and cargo outdated for all project workspaces.

What I ultimately what from this task is to make reviewing the changes to the upgraded packages as painless as possible.

I would like some feedback on my plan to make a cargo plugin that allows the user to bulk review all package upgrades.

My idea is to present each potential package upgrade interactively where I can see simple stats on the upgrade, see the change log entries added since my version and show me a diff of the code.

This plugin is taking the attitude that you can't trust anyone else's code. But we'll make it as painless as possible to review the incoming code changes.

I can imagine this being part of a broader social review system, but... that's getting ahead of myself.

feedback is much appreciated.
