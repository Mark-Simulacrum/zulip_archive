Topic: Memory safety bugs in Rust std

Yechan Bae (Apr 08 2021 at 23:08):

  1. Right now, RustSec only tracks Rust std bugs that have CVE numbers assigned. Unfortunately, this makes memory safety bugs in the Rust standard library underrepresented than a bug in a toy crate on crates.io.
  2. In the last attempt (#561), we decided to request a CVE number before creating a RustSec advisory to be consistent with other std bugs. This seems to be much slower than I expected. I requested CVE number through MITRE form but haven't heard back anything for 3.5 weeks.
  3. In comparison, id-map was reported to RustSec on April 2nd and got CVE numbers on April 7th, in 5 days. I don't know who it is, but some CVE authority seems to be monitoring RustSec bugs and filing CVEs for them.
  4. Can we do the same thing for std bugs?
