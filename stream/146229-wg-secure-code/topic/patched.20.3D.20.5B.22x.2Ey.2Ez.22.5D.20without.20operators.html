We have an advisory that does not declare any operators on patched versions: RUSTSEC-2021-0048. It lists patched = ["1.0.9"]
Under both the old and the new version matching logic this means "anything semver-compatible with 1.0.9", so 2.0 and onwards will be reported as vulnerable.
I suppose I should just patch this particular advisory to list >= 1.0.9, but what should we do in such cases going forward?
The options that occur to me are either treating it as an implicit >= or prohibiting this kind of ambiguous specification. Thoughts?
