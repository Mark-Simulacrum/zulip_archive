Topic: `libffi-rs` is unsound

Monadic Cat (Nov 06 2021 at 18:33):

Hi! I mentioned this in the Rust community Discord and was told I should post here.

The libffi crate, which provides bindings to the libffi library, is unsound with respect to Send and Sync, in that it, through only safe APIs, erases all auto traits on closures.
It also erases the mutability required to call them, in that you can call a ClosureMutN via shared references.
