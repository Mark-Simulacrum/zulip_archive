Hey all! :wave: So I regularly forget to release new versions of things or even what state things are in, so I had a half-baked thought of trying to process our way out of it a bit by:

  • Having a sync up on where a bunch of libs are at at the start of Rust's 6-week release cycle
  • Automating publishing those libs to crates.io by PRing main to release
  • Have a release-a-thon for all the libs towards the end of Rust's 6-week release cycle

What do you all think? Would anybody be interested in meeting a bit more regularly to catch up on all the libraries and RFCs and things we care about?
