Is there a particular reason why a const fn len() -> usize hasn't been implemented for arrays, or is it just that nobody has gotten to it yet?
While in most cases it can be taken from the type in a generic function, such a function is useful for:

  • Any sort of macro that requires knowing the length of the array that it's operating on
  • Avoiding having to "pollute" the namespace with unnecessary generic variables
