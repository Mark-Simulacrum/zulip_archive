Topic: retiring rust-icci?

mw (Jan 25 2019 at 09:51):

While initially developing incremental compilation, @nikomatsakis and I created cargo-incremental, a tool that allows to reply the git history of some project and compile each commit incrementally against the previous one. Built on that I created rust-icci, a test suite of sample projects that verifies that incremental compilation yields correct code and doesn't crash. It has helped us a lot in the early development phases, catching bugs not covered by our test suite.

However, it's also a maintenance burden (these are old code bases that need adaption in order to keep compiling with newer Rust versions) and apart from me, nobody is really looking at the results. Since incremental compilation is the default for debug and check builds these days, so I think it gets enough testing in the wild and rust-icci has outlived its usefulness.

Does anybody have any objections to my turning it off?
