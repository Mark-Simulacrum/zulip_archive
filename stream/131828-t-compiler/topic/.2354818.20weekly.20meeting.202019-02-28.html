matklad (Feb 28 2019 at 15:49):

We were thinking about what work to actually do, and two approaches seemed reasonable:

  • provide an end-to-end language server, which can do everything, but for a subset of the Rust langauge (ie, don't handle nasty macros, don't handle corner casees)

  • alternatively, we can focus on some specific fragments of compiler pipeline (line parsing, or name resolution), and try to extract them as a separate, production ready libraries
