Mark Juggurnauth-Thomas (Feb 15 2021 at 18:39):

One more thought I had about count: in the current proposal the default is to count the first depth, with the parameter adding more depths.
It might be more natural if the default was to count all depths, and with the parameter limiting the number of depths to that number of nestings. i.e. ${count(foo)} vs. ${count(foo, 1)}. This would allow ${count()} to do something consistent when the number of nestings changes, and cleans up the weird off-by-one where ${count(foo, 2)} counts into three nested repetitions.
Thoughts?
