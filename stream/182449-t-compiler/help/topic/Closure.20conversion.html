Can someone help me find the place where rustc lifts closure bodies into top-level functions (i.e. closure conversion)?

eholk (Jan 13 2022 at 21:26):

I was assuming this happens somewhere in the HIR -> MIR step, but maybe it actually happens in the AST -> HIR phase?

bjorn3 (Jan 13 2022 at 22:05):

Basically when you ask for the MIR body of <{closure type} as FnOnce>::fn_once (or one of the other Fn* methods depending on which are implemented) it will produce the MIR body of the closure. There is no explicit conversion to a top level function.

Noah Lev (Jan 13 2022 at 22:16):

Yeah, closures are explicitly represented in MIR. E.g., this

fn main() {
    let _x = |y: i32| { y + 1 };
}

turns into this:

// WARNING: This output format is intended for human consumers only
// and is subject to change without notice. Knock yourself out.
fn main::{closure#0}(_1: &[closure@src/main.rs:2:14: 2:32], _2: i32) -> i32 {
    debug y => _2;                       // in scope 0 at src/main.rs:2:15: 2:16
    let mut _0: i32;                     // return place in scope 0 at src/main.rs:2:23: 2:23
    let mut _3: i32;                     // in scope 0 at src/main.rs:2:25: 2:26
    let mut _4: (i32, bool);             // in scope 0 at src/main.rs:2:25: 2:30

    bb0: {
        _3 = _2;                         // scope 0 at src/main.rs:2:25: 2:26
        _4 = CheckedAdd(_3, const 1_i32); // scope 0 at src/main.rs:2:25: 2:30
        assert(!move (_4.1: bool), "attempt to compute `{} + {}`, which would overflow", move _3, const 1_i32) -> bb1; // scope 0 at src/main.rs:2:25: 2:30
    }

    bb1: {
        _0 = move (_4.0: i32);           // scope 0 at src/main.rs:2:25: 2:30
        return;                          // scope 0 at src/main.rs:2:32: 2:32
    }
}

fn main() -> () {
    let mut _0: ();                      // return place in scope 0 at src/main.rs:1:11: 1:11
    let _1: [closure@src/main.rs:2:14: 2:32]; // in scope 0 at src/main.rs:2:9: 2:11
    scope 1 {
        debug _x => _1;                  // in scope 1 at src/main.rs:2:9: 2:11
    }

    bb0: {
        return;                          // scope 0 at src/main.rs:3:2: 3:2
    }
}
