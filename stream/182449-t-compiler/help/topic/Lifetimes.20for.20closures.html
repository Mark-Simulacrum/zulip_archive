Ben Reeves (Dec 20 2021 at 03:01):

@Patrick Poitras I don't have a direct answer for you, but as a casual contributor, the approach I tend to take when trying to figure out "which part handles X" is as follows:

  1. Grep for fragments of the error message to find the code that produces that type of diagnostic. That should get you to the correct crate, which is half the battle.
  2. Follow the trail of breadcrumbs upward. Who calls that function that produces the diagnostic? Under what conditions? Which function produces the data that is used to decide whether or not the diagnostic should be produced?
  3. Recompile rustc with additional debug/logging and run it with RUSTC_LOG on the minimal example to help you better figure out what data is influencing the compiler's decisions: https://rustc-dev-guide.rust-lang.org/tracing.html
