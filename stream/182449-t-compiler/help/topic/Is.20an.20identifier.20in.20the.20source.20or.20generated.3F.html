Topic: Is an identifier in the source or generated?

Will Crichton (Dec 02 2021 at 21:23):

Let's say I have a function like:

fn ex() {
  let v = [0];
  for _ in v.iter() {}
}

This generates the HIR:

{
    let v = [0];
    {
        let _t =
            match #[lang = "into_iter"](v.iter()) {
                mut iter =>
                loop {
                    match #[lang = "next"](&mut iter) {
                        #[lang = "None"] {  } => break ,
                        #[lang = "Some"] { 0: _ } => { }
                    }
                },
            };
        _t
    }
}

And the MIR body has the following source scopes:

 scope 1 {
    debug v => _1;                   // in scope 1 at src/plateau.rs:17:7: 17:8
    let mut _6: std::slice::Iter<'_#13r, i32>; // in scope 1 at src/plateau.rs:18:12: 18:20
    scope 2 {
        debug iter => _6;            // in scope 2 at src/plateau.rs:18:12: 18:20
    }
}

When I have a MIR body, is there an easy way to determine that v exists in the source while iter was generated by the compiler?

cjgillot (Dec 02 2021 at 21:31):

This information should be in the span. See Span::is_desugaring and Span::desugaring_kind.
