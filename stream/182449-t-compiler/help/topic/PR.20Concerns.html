Would it be best to address these issues in the PR description, or in another comment.

Both are ok.
PR description goes into the merge commit message when the PR is merged though, so the PR description may need to be updated before the merge if some issues are addressed.

Additionally, should I mark the PR as a draft until those concerns are resolved?

We have labels like S-waiting-on-author, S-waiting-on-review and others so GitHub's draft functionality is entirely redundant.
