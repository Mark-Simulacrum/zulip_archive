What exactly is an allocator shim?

bjorn3 (Sep 22 2021 at 16:47):

@Timothy Maloney When compiling a crate it isn't yet known if the executable or dynamic library will use an allocator provided by #[global_allocator] or the default libstd allocator. For this reason they will contain calls to functions like __rust_alloc. The allocator shim is then created when compiling the executable or dynamic library which provides this function and dispatches it to either #[global_allocator] or the default libstd allocator.
