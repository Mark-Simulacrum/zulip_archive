is_global should pretty much always be used unless you're inside of predicates/generics_of

lcnr (Nov 25 2021 at 10:36):

is_global can cause query cycles inside of these queries as it relies on the predicates of the parent of unevaluated constants) while is_known_global returns false if the type contains an unevaluated constants with unknown substs, even if these are actually empty
