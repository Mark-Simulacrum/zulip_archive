Topic: Testing error index markdown files

Kevin Tran (Jan 14 2021 at 10:32):

How do I test the code examples in the error index markdown files?

Eric Huss (Jan 14 2021 at 15:29):

./x.py test src/tools/error_index_generator

If you have already run the command above: ./x.py test --keep-stage=0 --keep-stage=1 src/tools/error_index_generator will help avoid rebuilding the compiler each time you edit the markdown.

Personally I would iterate in a temporary local project (or on play.rust-lang.org if you prefer), and only run the full tests just before opening the PR.
