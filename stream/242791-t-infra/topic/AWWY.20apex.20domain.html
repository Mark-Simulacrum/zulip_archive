Hello! 2 years ago I opened a ticket to get the apex domain working with AreWeWebYet. Recently, it has moved to the rust-lang org, and the domain's registrar has changed from Ghandi to AWS. I'm assuming that registrar switch means the domain is now managed by the rust team too. When the default domain is set to the www subdomain with GitHub Pages, the apex domain will not redirect properly (and vice-versa). The best solution would be adding a CNAME/ALIAS record (idk what AWS calls it) to the apex domain pointing towards the www subdomain. There's a bit more information at https://github.com/rust-lang/arewewebyet/issues/129#issuecomment-740371721.
