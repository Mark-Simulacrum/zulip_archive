Topic: Potential github bot for starting a Zulip thread?

Manish Goregaokar (Dec 05 2021 at 04:02):

I feel like in many of my projects it's common to cc @rust-lang/teamname to get wider input in a github thread. This is pretty great, but what i've found is that it ends up happening pretty often and ends up subscribing me to more threads than I want to be on.

There are some cases where you actually do have to ping the entire team (when you want approval for something, etc), but sometimes when you "just want wider input" the better thing to do is to start a Zulip thread. It's also a better platform for such discussions in general.

I wonder if there's benefit in e.g. having something like @triagebot zulip-thread <topic name> so that it's super easy to create zulip threads that link back to the issue (and ideally duplicate the text of your comment after the @triagebot command), and get a comment on the issue linking to the zulip thread. I feel like this would be super useful but I'm not sure which bot to plug it into.
