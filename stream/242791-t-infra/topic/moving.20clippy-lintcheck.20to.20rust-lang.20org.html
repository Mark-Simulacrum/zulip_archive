Hi, how much free CI capacity does the rust-lang org have left?
At clippy we have a new tool called lintcheck to run clippy on a fixed configurable set of crates and compare clippy outputs between clippy versions (it's a bit like crater but dumps all the clippy warnings into the repo afterwards for diffing). This can help us spot unexpected lint output changes (FPs/FNs) that appear in code in the ecosystem.

Right now I have a prototype at https://github.com/matthiaskrgr/clippy-lintcheck which runs every 12 hours, each run takes around 35-45 minutes of ci time (it only needs a single runner though).
https://github.com/matthiaskrgr/clippy-lintcheck/runs/2293318284?check_suite_focus=true

At the last clippy meeting we wondered if we could move the repo to the rust-lang org. :)
