jsha (Jan 31 2022 at 20:54):

I'd like to make a two-part proposal about the redirect from doc.rust-lang.org to rust-lang.org/learn:

I'd like to make it not a redirect, so it stays on the doc.rust-lang.org domain (but the page design could stay the same).
I'd like to add a search box in a moderately prominent place, which goes to https://doc.rust-lang.org/std/index.html?search=foo.
The small reason is: sometimes I find myself typing in the base domain, figuring I can navigate docs from there, or edit the URL if needed. It's jarring to get taken off-domain.

The big reason is: I'd like for users of the Rust documentation to get the nice browser integration where you type, e.g. "Rust " and it offers "search Rust Documentation" as an option. Based on research, it seems this was possible to get based on OpenSearch, but now the only way to get it is via browser heuristics, but those are fairly well documented. For Chrome, to get this experience, there needs to be a search box on the root page of a hostname (and the user needs to interact with it).

Does this sound good / feasible?
