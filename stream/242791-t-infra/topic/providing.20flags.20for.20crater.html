Topic: providing flags for crater

Jacob Pratt (Jan 20 2022 at 22:51):

I think this is the correct stream. Feel free to move if it's not.

Should we have a way to request crater provide certain flags when running tests? While features are supposed to be additive (which would let us pass --all-features), some crates have mutually exclusive features. I think it would be great to be able to tell crater what flags we need. Personally, the time crate requires --all-features to be passed. I currently (wrongly) omit tests from crates.io, but even if I didn't it would still fail for this reason. This was noted in this issue by @Ben Kimock (Saethlin).

Having a way to provide crater certain flags wouldn't be unprecedented. docs.rs, something not strictly tied to the language, (uses metadata in Cargo.toml for this purpose](https://docs.rs/about/metadata).
