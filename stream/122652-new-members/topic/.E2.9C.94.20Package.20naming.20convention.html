Laurențiu (Jul 26 2021 at 11:14):

If you mean snake_case vs. kebab-case, both are quite popular. I think the latter looks nicer, but can lead to some ambiguities that are generally handled well. Adding rust_ or rust- or -rs, it's generally frowned upon because it's already clear you're in Rust and typing meta_vars_rs::foo() gets old pretty quickly.

I see it like "crate name is meta-vars, but the Rust syntax requires you to type it as meta_vars"

bjorn3 (Jul 26 2021 at 11:52):

The package name is meta-vars, but the crate name is meta_vars. The crate name doesn't even have to be similar to the package name. Note that crate is a rustc concept meaning a single compilation unit, while package is a cargo concept meaning a collection of crates where at most one is a library, with any number of binaries, examples and tests.
