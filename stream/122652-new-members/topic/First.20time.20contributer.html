Edward Nicholes Jr. (Feb 26 2022 at 06:57):

Hey everyone. I was working on rustlings on the from_into.rs exercise:
https://github.com/rust-lang/rustlings/blob/main/exercises/conversions/from_into.rs

My solution resulted in this implementation of From:

impl From<&str> for Person {
    fn from(s: &str) -> Person {
        if s.is_empty() {
            return Default::default();
        }

        let mut split = s.split(",");

        let name: String = split.next().unwrap_or("").into();
        if name.is_empty() {
            return Default::default();
        }

        split.next()
            .and_then(|age| age.parse().ok())
            .and_then(|age| {
                match split.next() {
                    Some(_) => None,
                    None => Some(Person { name, age }),
                }
            }).unwrap_or(Default::default())
    }
}

But I though that instead of this...

split.next()
    .and_then(|age| age.parse().ok())
    .and_then(|age| {
        match split.next() {
            Some(_) => None,
            None => Some(Person { name, age }),
        }
    })
    .unwrap_or(Default::default())

... it would be nice to be able to do this ...

split.next()
    .and_then(|age| age.parse().ok())
    .and_then(|age| split.next().not(Person { name, age }))
    .unwrap_or(Default::default())

I figure this fits nicely along with the boolean like methods on Option like and, or, and xor. All we're missing is not

I created a PR and wanted to know what the rules were to submitting a PR to rust project.
https://github.com/rust-lang/rust/pull/94393
