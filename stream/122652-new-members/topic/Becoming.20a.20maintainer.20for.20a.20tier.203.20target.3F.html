Topic: Becoming a maintainer for a tier 3 target?

Ryan Zoeller (Dec 16 2021 at 00:05):

What's the process for becoming a maintainer for an existing tier 3 target? I'm not aware of an existing maintainer for DragonFly BSD (please point me to the docs if I've missed something), and I'm willing to try to fill that role.

Josh Triplett (Dec 16 2021 at 00:16):

@Ryan Zoeller There isn't a well-defined process for that, but roughly speaking, for any target that pre-dates the target tier policy, I'd send a PR adding target-specific documentation that lists yourself as the maintainer, and in the pull request description, make a fresh proposal as if you were proposing a new tier 3 target, checking off all the policy items.

Josh Triplett (Dec 16 2021 at 00:17):

That serves the dual purpose of giving us updated policy information and target documentation for an existing target and showing enough commitment to working on the target to help people feel comfortable adding you as a maintainer.

Josh Triplett (Dec 16 2021 at 00:18):

(In the PR, you'd note that the target already exists, but doesn't have a maintainer on file, and you'd like to become the maintainer.)

Jubilee (Dec 16 2021 at 00:20):

Also would help answer the "wait, why do we have this?" and "would anyone object if we nuked this code?" questions in the future
