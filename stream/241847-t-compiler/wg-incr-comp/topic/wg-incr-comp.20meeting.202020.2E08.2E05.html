cjgillot (Aug 06 2020 at 06:31):

I have a few commits there and there on the query system. My goal here is to allow decentralised query declarations into salsa-like query databases. There are a few design questions hidden in there:

  • should there be a dyn DepGraph or a split per-database DepGraph?
  • how to handle code that lists all queries (eg. stats)?
  • how to handle the on-disk cache? (more difficult: if both lists all queries, and fetches data for any one of them)

I will try and make a PR and a more elaborated document when I get the time.
