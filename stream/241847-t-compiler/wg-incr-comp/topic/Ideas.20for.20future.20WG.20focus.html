Topic: Ideas for future WG focus

mw (May 10 2021 at 12:59):

In light of recent events I think the working group should focus on correctness and clean implementation for a while :) In particular:

  • simplify things where possible
  • add more documentation, especially about assumptions the system makes
  • add more sanity checking of assumptions the system relies on
  • find cases that
    • side-step dependency tracking and/or the query system
    • do something weird with result hashing
    • do something weird during serialization and/or deserialization,
  • add more regression tests
  • think about ways to auto-generate test cases / do fuzzing
  • think about ways to test the compiler against public git histories (in a robust way)
