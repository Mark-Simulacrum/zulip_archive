For those that use VS Code (or VS Codium) nightly, there's a new setting that is quite nice when working on the compiler. File nesting is experimental and lets you collapse multiple files into a single logical item automatically. Personally I have expansion disabled by default, as most of the time you don't need to see everything.

To enable:

  • Go to your settings (ctrl + comma)
  • Search for file nesting
  • Check the box for "File Nesting: Enabled"
  • Optionally uncheck the box for "File Nesting: Expand"
  • Add the following rule to "File Nesting: Patterns":
    • Item: *.rs
    • Value: $(capture).stderr, $(capture).fixed, $(capture).stdout, $(capture).*.stderr

This will collapse all stdout, stderr, and fixed files under the code for the UI test. It can be expanded like any directory.

Hopefully this is useful to somebody! I've had it enabled for a few days and certainly find it beneficial.
