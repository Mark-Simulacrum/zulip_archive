When we say a piece of code 'runs', does it mean "it compiles correctly, and when running, may or may not crash on unit tests", or do we mean "the code compiles + passes all unit tests" ?////

Lokathor (May 10 2020 at 19:03):

Compiles and executes.

Passes unit tests or "doesn't crash" or other high standards aren't generally part of "runs".

Lokathor (May 10 2020 at 22:44):

Yes, I would say that:

  • "builds" / "runs" is "cargo build works without errors, and you can start the program". Since we say things like "when i run it, the program crashes", obviously we've accepted that code which "runs" isn't bug free.
  • "passes tests" is when all your unit tests and doc tests pass (which might not be enough test coverage of course).
  • "works" is when it actually can be demonstrated to do the job it's supposed to do in at least some portion of the possible situations it needs to handle. Example: "it works for u8 but not u16."
