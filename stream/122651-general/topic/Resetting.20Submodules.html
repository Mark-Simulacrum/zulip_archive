Topic: Resetting Submodules

Ibraheem Ahmed (Oct 09 2021 at 14:40):

In a PR I submitted (#88502) a bunch of changes for the llvm and rust-analyzer project got pulled in, and I can't seem to reset them. I tried git submodule foreach git reset --hard, and I tried manually resetting to the hash that master was on, but neither worked.

Eric Huss (Oct 09 2021 at 14:57):

@Ibraheem Ahmed Removing an accidental submodule is almost the same as removing any accidental file. There are multiple ways to do it, but for your case, here is an easy one:

  1. git reset --soft HEAD^
    This will reset your working directory to just before your commit.

  2. git restore --staged src/llvm-project src/tools/rust-analyzer
    This will unstage those changes.

  3. git submodule update
    This will restore the submodules to the correct commit.

  4. git commit
    Re-enter your original commit message.

  5. git push -f
    You'll need to force push now that you've modified the original commit.
