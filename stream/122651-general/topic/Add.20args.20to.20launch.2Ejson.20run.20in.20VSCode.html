Topic: Add args to launch.json run in VSCode

Benjamin Mordaunt (Apr 06 2022 at 00:10):

Hey, I'm trying to add launch command-line arguments to a launch target in launch.json, but it doesn't appear to actually apply them (my program complains that the arguments are missing).
The launch config looks like this:

{
            "type": "lldb",
            "request": "launch",
            "name": "Debug executable 'myexec'",
            "cargo": {
                "args": [
                    "build",
                    "--bin=myexec",
                    "--package=myexec"
                ],
                "filter": {
                    "name": "myexec",
                    "kind": "bin"
                }
            },
            "args": ["--my-awesome-arg"],
            "cwd": "${workspaceFolder}"
        }

--my-awesome-arg is not acknowledged by the application, but it is if I run it manually. Any ideas?

Robert (Apr 06 2022 at 15:09):

Does it work if you add "--" and "--my-awesome-arg" to "args" under "cargo"?

bjorn3 (Apr 06 2022 at 17:32):

No, cargo build doesn't accept run args. Only cargo run does. cargo build is needed as cargo run doesn't allow running the produced executable inside the debugger.
