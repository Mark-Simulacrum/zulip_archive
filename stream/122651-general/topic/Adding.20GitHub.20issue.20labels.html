I didn't see an obvious place to do metadiscussion about GitHub issue stuff, so here we go. I think we might want to do some fiddling with issue labels. And as a reminder, people can set issue labels via comments in an issue with the following syntax:

@rustbot label: +{LABEL_A}, -{LABEL_B}

where {LABEL_A} and {LABEL_B} can be any two existing labels.

Anyways, we didn't have a label for A-array so I figured I might as well go ahead and make one, as people were using A-slice for it instead, and const generics has kinda made them much more clearly their own distinct thing, so I figured they should get one.

Noah Lev (Oct 06 2021 at 21:53):

The way I think of A- labels is you have a "top-level" label like A-linkage and then, optionally, finer-grained labels like A-link-dead-code. Then, if you want to find all linkage issues, you can search for A-linkage, but if you want to find just -C link-dead-code issues, you can search for A-link-dead-code (or A-linkage + A-link-dead-code).

Noah Lev (Oct 06 2021 at 21:54):

E.g., A-diagnostics is a top-level label and A-suggestion-diagnostics is a sub-label. (Although sometimes I and others use just A-suggestion-diagnostics since it is so common.)
