Tip for anyone who has this problem: I had this annoying thing where I modified content in a submodule, but then I couldn't stage the changes, so I wanted to just get rid of them. I tried a bunch of things, but eventually I solved the problem by running rm -rf src/tools/cargo and then running ./x.py (which I think runs git submodule update even if you don't pass a command like build or test).
