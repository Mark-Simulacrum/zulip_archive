Topic: Enforce NULL checks via types

Elichai Turkel (Apr 05 2022 at 15:38):

Is there currently a way to enforce NULL checks on pointers via types without opting into provenance?
i.e.

pub unsafe extern "C" fn func(buf_ptr: *const u8, buf_len: usize, out32: *mut [u8; 32]) {
    assert!(!out32.is_null());
    assert!(buf_len == 0 || !buf_ptr.is_null());
    let buf = (buf_len > 0).then(core::slice::from_raw_parts(buf_ptr, buf_len));
    // Do stuff
    out32.write(ret_val);
}

Instead I'd like to do something like:

pub unsafe extern "C" fn func(buf_ptr: Option<*const u8>, buf_len: usize, out32: Option<*mut [u8; 32]>) {
    let buf = (buf_len > 0).then(core::slice::from_raw_parts(buf_ptr.unwrap(), buf_len));
    // Do stuff
    out32.unwrap().write(ret_val);
}

That way I don't need to write all the asserts and make sure they match the required logic (some pointers are optional etc.)

but currently, this isn't ffi safe as it's 2 words, and Option<&u8> makes this function UB, and Option<NonNull<u8>> loses the const/mut annotations and makes everything mut :(
