Reaur (Nov 11 2021 at 02:20):

compiler not allow use variable after that variable captured with closure with 'move' keyword.
Important part is; closure is not used, only defined.
Closure should be lazy, it should not take ownership of captured values until called.

Example:
let my_vec = vec![1, 2, 3, 4];
let my_closure = move |x| my_vec.get(0).unwrap() == x;
println!("{:?}",my_vec ); // Error: Move occurs

It should be;
let my_vec = vec![1, 2, 3, 4];
let my_closure = move |x| my_vec.get(0).unwrap() == x;
println!("{:?}",my_vec ); // Should be OK
let res = my_closure(5); // closure is called, so take ownership of captured values
println!("{:?}",my_vec ); // Should be Error

panstromek (Nov 11 2021 at 11:03):

Whether or not this a good idea, it would be breaking change, with some subtle implications. Consider this code:

fn main() {
    let mut x = 5;
    let mut c = move || println!("{}", x);
    x += 1;
    c(); // currently prints 5, but after this change, it would print 6. Moving it up one statement would change the output to 5 again.
}

Charles Lew (Nov 12 2021 at 18:07):

A closure can be called multiple times, so your design will bring in a lot of complexity. If you need this behavior your can do it manually in this way:

Charles Lew (Nov 12 2021 at 18:07):

let my_vec = Some(vec![1, 2, 3, 4]);
let my_closure = |x| my_vec.take().get(0).unwrap() == x;
println!("{:?}",my_vec ); // ok now.
