Chris Denton (Jul 30 2021 at 13:15):

I'm having trouble navigating the history of some code that's been moved. I reach a dead end when I get to "File renamed without changes". There doesn't seem to be a way to jump to the file before it was renamed?

Eric Huss (Jul 30 2021 at 13:36):

Unfortunately GitHub's history view does not follow renames. There are a few options:

  • Use the Blame view which follows renames. This is what I usually use, though of course it doesn't show a linear history. I'm normally interested in a specific segment of code.
  • Click the rename commit, then click "parent", then browse to the old file location, then resume looking at its history.
  • Look at the history locally with git (or your IDE if it has fancy git capabilities), which can follow renames.

Eric Huss (Jul 30 2021 at 13:38):

Also, this won't help you with renames, but I just wanted to share https://github.githistory.xyz/ which I find as a nice way to browse through history. Unfortunately due to all the renames in the rust-lang/rust repo, it quickly hits a dead end. :frown:
