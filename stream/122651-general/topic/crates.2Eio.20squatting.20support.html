Using an automated tool to claim ownership of a large number of package names is not permitted. We reserve the right to block traffic or revoke ownership of any package we determine to have been claimed by an automated tool.

Chris Jester-Young (Dec 12 2020 at 23:11):

So, I read through Manish's pre-RFC. That's certainly one way to solve the problem I have in mind, in that you could have digest/md5 and digest/sha1. But the default package names of digest_md5 and digest_sha1 is a bit gross.

When I specify @rustcrypto/md5 or @rustcrypto/sha1 in my proposal, I am expressing a disinterest (at least by default) in using the top-level md5 and sha1 crates. I already do this in my Cargo.toml files, using sha1 = { package = "sha-1" }. This already signals my disintent to use the top-level sha1 crate, and that anywhere I use sha1 in my code, I only want to use the genuine crate as published by RustCrypto.

That's the use case I'm trying to facilitate. :-)
