I'm having an issue with implementing a trait on a type where the same trait is implemented for &mut Self:
https://play.rust-lang.org/?version=stable&mode=debug&edition=2021&gist=beadbe8b8ad1bf99e44227d5da5b3c83

trait Foo<T> {
    type Output;
    fn bar(self, value: T) -> Self::Output;
}

struct Struct;

impl<'r> Foo<usize> for &'r mut Struct {
    type Output = ();
    fn bar(self, value: usize) -> Self::Output {
        println!("<&mut Struct as Foo<usize>>::bar()");
        // Complicated logic
    }
}

impl<T> Foo<T> for Struct
where
    for<'r> &'r mut Struct: Foo<T>
{
    type Output = <&'what_do_i_put_here mut Struct as Foo<T>>::Output;

    fn bar(mut self, value: T) -> Self::Output {
        println!("<Struct as Foo<T>>::bar()");

        let out = (&mut self).bar(value);

        // Some finishing step

        out
    }
}

fn main() {
    <Struct as Foo<usize>>::bar(Struct, 3);
}

It seems I need the hrtb on the implementation to avoid the error about 'r being unconstrained, but then there's no way to refer to the associated type.
