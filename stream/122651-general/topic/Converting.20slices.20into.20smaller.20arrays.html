Topic: Converting slices into smaller arrays

Arthur Cohen (Feb 01 2022 at 20:59):

Hi everyone,

A friend of mine stumbled upon a case where they wanted to convert a large slice into a smaller array. The length of the slice is more than the length of the target array, which according to the check here fails.

The workaround for this is to do the same unsafe operation by hand, which works for simple types but might pose issues for more complex ones I assume?

let slice = [1, 2, 3, 4, 5, 6];
let small_array =  unsafe { &*(slice..as_ptr() as *const [u32; 2]) };

In the doc, it is explained that converting a vector to an array requires that both source and destination be of the same length: https://doc.rust-lang.org/std/primitive.array.html#method.try_from-3. In the case that we are only looking for a prefix of the vector, we can call the truncate method on it first. However, this method is not available on slices.

Is there any reason why the aforementioned behavior is discouraged? Any way to resort to it without explicitly going through a Vector?

Thanks!
