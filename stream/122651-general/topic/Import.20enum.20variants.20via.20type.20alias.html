Topic: Import enum variants via type alias

João M. Bezerra (Aug 23 2021 at 21:56):

This works

use Option::*;

But this does not work

type MyOption<T> = Option<T>;
use MyOption::*;

Because of this limitation, I think that using type aliases of enums is always bad, specially bad if it's exposed by the crate's public API.

Maybe this should be supported as type aliases can be useful for working with generic types?
