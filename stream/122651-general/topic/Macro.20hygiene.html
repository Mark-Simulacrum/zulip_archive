// https://play.rust-lang.org/?version=stable&mode=debug&edition=2021&gist=3f08913c5b0a25497722ca9f88005961

macro_rules! fun {
    ( $fun:ident(|$($arg:ident: $arg_ty:ty),*| { $ex:expr }) ) => {
        pub fn $fun( $($arg: $arg_ty),* ) {
            let ex = $ex;
            dbg!(ex);
        }
    }
}

fn main() {
    fun!(some_fun(|a: i32, b: i32| { a + b }));
    some_fun(5, 6);
}

In my limited option, a + b should not be aware of a: i32, b: i32 since they separated units (macro hygiene?) but everything is working somehow :shrug:

Aaron Hill (Dec 19 2021 at 14:04):

When you use a metavariable, it keeps the original hygiene

Aaron Hill (Dec 19 2021 at 14:05):
