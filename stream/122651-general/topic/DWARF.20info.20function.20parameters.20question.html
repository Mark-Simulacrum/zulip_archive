Topic: DWARF info function parameters question

vn-ki (Sep 19 2021 at 20:15):

Not sure where I should ask this, so asking it here. I was trying to parse some DWARF info on a binary.

#include <stdio.h>

int fact(int x) {
    if (x < 1) {
        return 1;
    }
    return x * fact(x - 1);
}

int main() {
    printf("%d", fact(10));
}

this is the binary,

The formal parameter info with readelf

 <2><cc>: Abbrev Number: 11 (DW_TAG_formal_parameter)
    <cd>   DW_AT_name        : x
    <cf>   DW_AT_decl_file   : 1
    <d0>   DW_AT_decl_line   : 3
    <d1>   DW_AT_decl_column : 14
    <d2>   DW_AT_type        : <0x58>
    <d6>   DW_AT_location    : 2 byte block: 91 6c      (DW_OP_fbreg: -20)

with ddbug-parser (https://docs.rs/ddbug_parser/0.3.0/ddbug_parser/struct.Parameter.html)

param.frame_location() = [FrameLocation { offset: -20, bit_size: Size() }]
param.registers() = []

But looking at the generated code, it definitely seems to be passed through rdi

    1168:       bf 0a 00 00 00          mov    edi,0xa
    116d:       e8 c7 ff ff ff          call   1139 <fact>

Am I misinterpreting the location here? How can I get where the variable is stored?

bjorn3 (Sep 19 2021 at 21:45):

The DW_AT_location shows where the parameter will be stored by fact after the call. It doesn't show how it is passed as argument to fact.
