Topic: Result -> bool, logging Err

Tim Siegel (Dec 11 2021 at 03:11):

Hi, I have result: Result<u32, String> at the end of a function that needs to return bool (result.is_ok()). I want to log the Err before returning, but it is messy:

fn foo(input: &str) -> bool {
    let result: Result<u32, String> = input.long().computation();
    if result.is_err() {
        // Returning Result to caller would be nice, but the API is fixed
        #[allow(unused_must_use)]
        writeln!(stderr(), "Failure({})", &result.clone().unwrap_err());
    }
    result.is_ok()
}

It works, but the compiler isn't happy:
