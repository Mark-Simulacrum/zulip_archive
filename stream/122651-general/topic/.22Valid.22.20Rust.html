matt1992 (Oct 19 2021 at 02:41):

I'll note that the _block in the if let Err(_block) pattern doesn't pattern match against the value in the previously declared _block variable, it shadows the variable.
Using clippy to detect it:

#![deny(clippy::shadow_unrelated)]

use std::io::{Error, ErrorKind};
use std::net::TcpListener;

fn main() {
    let listener = TcpListener::bind("127.0.0.1:2300").unwrap();
    listener.set_nonblocking(true).unwrap();
    let _block = Error::from(ErrorKind::WouldBlock);
    if let Err(_block) = listener.accept() {}
}

error: `_block` shadows a previous, unrelated binding
  --> src/main.rs:10:16
   |
10 |     if let Err(_block) = listener.accept() {}
   |                ^^^^^^
   |
note: the lint level is defined here
  --> src/main.rs:1:9

1  | #![deny(clippy::shadow_unrelated)]

note: previous binding is here
  --> src/main.rs:9:9
   |
9  |     let _block = Error::from(ErrorKind::WouldBlock);
   |         ^^^^^^
