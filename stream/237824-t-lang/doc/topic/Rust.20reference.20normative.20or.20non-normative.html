Yea, I agree with Niko. I think maybe some clarifications would be:

...is currently not normative.

and the second part:

We also want the reference to be up to a level of quality where it can eventually be considered normative. If you see anything this is wrong or incomplete, please also file an issue.

I'm not exactly sure if that is the right way to phrase it, since the reference is so far from being a full specification that it isn't really clear if that is the end goal. Like an eventual formalism might look completely different. But the current track is that it strives to be a full and correct description of the language. I feel like sometimes people get hung up on the word "normative", and I would be fine just removing it.
