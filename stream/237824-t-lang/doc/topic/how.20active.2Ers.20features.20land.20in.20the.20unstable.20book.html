oliver (Oct 03 2020 at 18:46):

Is there value in documenting some of these features which are defaulting to the
generic output?

Eric Huss (Oct 03 2020 at 19:07):

Like the stub pages? It depends on the feature. If it just has a link to the tracking issue, and the tracking issue contains enough documentation, there probably isn't a strong reason to add anything. If it's just a new API in the standard lib, the docs are in the standard lib (and should be relatively easy to find via the tracking issue). If there aren't any docs in the tracking issue, then it probably should be added either in the tracking issue or in the unstable book. If it is the "internal only" stub, I would like to at least see something (a link to the relevant section of the rustc-dev-guide, or a brief description, or a link to the code if it is documented there).
