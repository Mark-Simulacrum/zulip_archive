Topic: string continuation

Eric Huss (Jun 09 2021 at 18:56):

A question came up on https://github.com/rust-lang/reference/pull/1042. The behavior of \ at the end of a string line has surprising behavior in that it consumes all whitespace. This is different from other languages like Python where it will only consume a single line. It is not clear if this was intended, since the reference was explicitly written to say it is only a single line, and the rustc tests only check for a single line.

It has been this way since it was introduced about 10 years ago: https://github.com/rust-lang/rust/commit/df9cf0be9b898bc3432d444826ff05d6d0e22f97

The question is, should the reference be changed to document how it actually works? (I think the answer is yes.) Is anyone else surprised by this?
