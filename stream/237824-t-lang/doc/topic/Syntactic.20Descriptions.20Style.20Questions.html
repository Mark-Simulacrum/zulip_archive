Topic: Syntactic Descriptions Style Questions

isHavvy (Feb 28 2021 at 06:43):

We have paragraphs that are of the form:

The syntax for XYZ is a `some token`, and then a ABC, the ABCN, and then a PQR, the PQRN, and then `some other token`.

I have two questions about this:

  1. For expressions that contain subexpressions, we call them "operands" but AFAIK, only the reference uses the term "operand" since we're the only place that needs to be so precise with our terminology. Should we still call the syntactic element an expression, but use the name operand? E.g.
The syntax for a parenthesized expression is a `(`, then an operand, the *enclosed operand*, and then a `)`.

Would it be more readable if instead, it said "expression" but still named it an operand?

The syntax for a parenthesized expression is a `(`, then an expression, the *enclosed operand*, and then a `)`.
  1. Should we put "called" before the name? Using the same example as before:
The syntax for a parenthesized expression is a `(`, then an operand, called the *enclosed operand*, and then a `)`.
