flip1995 (Feb 04 2022 at 11:09):

For Clippy, blocking stabilization because Clippy isn't ready is definitely not a real option. If that were the case we wouldn't have async-await to this day.

For the let_else feature we already found and fixed a two bugs in Clippy.

Though for Clippy it is useful if a features lives a bit longer in nightly, so bugs in lints because of new features don't get into stable. A recent example of this is the implicit named arguments feature, which broke our useless_format lint. This bug got into stable, but with 1.58.1 we were lucky and could backport the fix. I think this feature was for a too short of a time in nightly, so that no one was using it, who also hit the Clippy lint.

So blocking stabilization because of Clippy is not an option. But possibly waiting an addition nightly cycle might be useful in some cases.
