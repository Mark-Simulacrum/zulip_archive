Leo Le Bouter (Mar 13 2021 at 00:20):

Hello!

I see it is possible to execute some tests by their name, but is it possible to exclude some tests while running all other known ones?

Thank you

Eric Huss (Mar 13 2021 at 00:24):

If you're referring to Cargo integration tests, there isn't any option to exclude. If possible, you can use glob patterns.

If you're referring to test function names, the test runner supports the --skip option (see docs)

Leo Le Bouter (Mar 13 2021 at 00:27):

@Eric Huss thanks for answering, I mean when using cargo on some project: cargo test, how to exclude some tests. So I use RUSTFLAGS to set the --skip option?

Eric Huss (Mar 13 2021 at 00:27):

The arguments go after the --. For example, cargo test -- --skip=testname
