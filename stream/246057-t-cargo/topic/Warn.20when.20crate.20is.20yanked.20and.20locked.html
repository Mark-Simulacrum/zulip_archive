I've seen a lot of broken builds because some dependency made an unnoticed breaking change in a patch version, then this version got stuck in a Cargo.lockand then some other dependency couldn't work because of the breaking change. I thought maybe cargo can emit a warning about yanked dependencies? Something like this:

warning: crate `A#x.y.z` is yanked, but it's locked in `Cargo.lock`, try running `cargo update`
