Topic: What's meant to go in .cargo-checksum.json?

gspr (Nov 15 2021 at 13:38):

When dealing with vendored crates, Cargo complains if it doesn't find the right content in a given crate's .cargo-checksum.json. I've pieced together that this file should contain a key "package" with a value equal to the SHA256 sum of the crate's .crate file. But I can't find documentation anywhere as to what that .crate file is supposed to be. It's some tarball of the crate's contents, right? But in what order, with what tar headers, etc?

gspr (Nov 15 2021 at 13:39):

Is this documented anywhere? Or, alternatively, can I instruct cargo to ignore the checksum?

bjorn3 (Nov 15 2021 at 13:42):

A .crate file is created by cargo package. When dealing with vendored crates using cargo vendor you shouldn't modify the crate source. .cargo-checksum.json exists to be more certain that the crate source is identical to the crates.io version. If you want to modify the crate source you should use a [patch.crates-io] section in Cargo.toml instead.

gspr (Nov 15 2021 at 13:46):
