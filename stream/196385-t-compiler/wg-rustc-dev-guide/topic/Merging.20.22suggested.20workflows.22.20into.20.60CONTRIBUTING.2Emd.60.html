Hi all. Recently, I've had the urge to expand the "suggested workflows" of the dev-guide and merge it into a cleaned-up CONTRIBUTING.md. I laid out these goals in the docs stream on discord

I've long wanted to make onboarding easier by giving a very detailed "walkthrough" for each of the ways of contributing in-tree that are listed in CONTRIBUTING.md. Specifically, I want to provide an exact series of ./x.py and git commands that will are needed to, for example, check out, modify and build the standard library documentation.
Do others think this would be desirable and (more importantly) worth the maintenance burden?

Even just telling people what parts what things are going to be slow ahead of time (initial checkout, first build of LLVM if not using the system one) would make things less daunting for first-time contributors.
