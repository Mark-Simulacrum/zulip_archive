Joshua Nelson (Oct 27 2020 at 22:31):

so I don't forget: we did not change much of the dev-guide, but the two main takeaways were
a) x.py is bad at detecting the target platform on windows (it thinks you have GNU if uname works, even if you don't have a GNU compiler). This could be fixed by looking at rustup default.
b) x.py setup doesn't run until the rust code runs, which means if your toolchain is broken you don't see it at all. This is a little harder to fix - maybe it could be moved into python? I'd prefer to keep things in rust as much as possible, though ...
