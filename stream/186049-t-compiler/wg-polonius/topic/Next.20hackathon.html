Just let me know what I can do to be maximally helpful.

Part of the reason I was interested in Souffle was to experiment with a basic block-aware version of live/init, which needed arithmetic predicates, dense relations (radix tries), and disjunction/inlining. I wanted to be able to run the rustc test suite in a reasonable amount of time and evaluate which features should be ported into datafrog. Obviously liveness is no longer needed for polonius.next, so this is less important, but it still interests me. I thought it would be useful for other types of prototyping as well, since you don't need to do variable indexing yourself.
