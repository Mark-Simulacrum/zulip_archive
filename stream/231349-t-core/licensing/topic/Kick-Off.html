Florian Gilcher (Jun 15 2020 at 10:28):

Here comes the summary. For reference: https://hackmd.io/r0QBWkvMSae43pfxNdTf6A?both

We discussed some workflow things. As much as possible, outside of meeting notes, we'd like to keep it to Zulip+gh issues, if that's fine by you.

We've identified three main topics:

  • Fixing known issues and better documentation (mostly rewriting the copyright file)
  • Help to Rust developers and teams to improve the licensing situation
  • Help to Rust users to comply with Rust licensing, through better documentation

The linked document has a number of GH issues currently open.

Felix and I tried to find a number of good issues to start working with:
1) bringing https://paper.dropbox.com/doc/rustc-licensing-guidelines--Ao~xMyRsp7cmHLrQtvQPS_eQAg-13Vo9fltFmrsNrFpddiMA over the finishing line. GH issue: https://github.com/rust-lang/compiler-team/issues/220. This document does not need to be perfect and substantial parts are written.
2) Checking out the rustc_apfloat issues currently: https://github.com/rust-lang/rust/issues/55993. This issue is a bit annoying, as the license of upstream has moved, and we're unclear about the current situation, that might be an issue where @Josh Triplett detailed license experience might help.
3) https://github.com/rust-lang/compiler-builtins/issues/319 is a tiny issue that can be worked through. We have someone at hand for a cleanroom implementation, if needed.

We did come up, but not nominate:

  • We need some categorisation of issues, such as "impacts compiler" or "impacts users", graded by severity.
  • compiler-builtins declares the wrong license: https://github.com/rust-lang/compiler-builtins/issues/307 (this is a bit bigger, but a good one to work through later)
  • Licensing help for users.
    * This has design space, especially around automation
  • Rewrite COPYRIGHT file
    * This is a huge task, not good as a starter task
  • Work through https://github.com/rust-lang/rust/issues/63232#issuecomment-517918295
    * This needs some planning on which to tackle first, but is also a good "next" task
