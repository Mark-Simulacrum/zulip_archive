Bram Geron (Apr 05 2022 at 21:19):

(continuing from #t-lang/wg-unsafe-code-guidelines > Problems with PVI - What optimizations does it break? )

Let's say we use an SMT solver. That makes everything easier to model than just SAT. Here's a sketch for a Miri-like checker.

Our world contains locations, which are each assigned a stack and values. Both stacks and values can mention tags. There is nondeterminism in these stacks and in the values. However, if we erase the tags out of values then they are the same in every nondeterministic world. This means that we can still execute the program deterministically — as long as we can figure out the erased value of every location.

Defining the SMT model

Let's reify the way in which the "spine" of values is always the same. We introduce a kind of "tag variables" 𝑡. Then a value becomes

𝑆𝑐𝑎𝑙𝑎𝑟 ≜ Pointer(𝑙, 𝑡) | 𝑧      where 𝑧 ∈ ℤ

(swapping out 𝑡 for t).

Stacks and items stay the same, and contain ordinary natural-valued tags.

In our SMT model, we introduce

  • for every location a tag variable 𝑡_𝑖 : ℕ
  • for every location a stack variable 𝑠𝑡𝑎𝑐𝑘_𝑙 : 𝑆𝑡𝑎𝑐𝑘

The scalar part of memory stays out of SMT-land, they do not get variables.

We write 𝑃((𝑡), (𝑠𝑡𝑎𝑐𝑘)) for a predicate over this model.

Execution

At runtime, we keep

  1. The program under evaluation. (expression + call stack, I guess)
  2. The set of live locations
  3. For every live location, its current scalar value — with tags replaced by tag variables as above
  4. A satisfiable SMT predicate over the model above.

To execute the program, execute the following in a loop.

(5) Use (1) to determine which operation to evaluate next.
(6) Let 𝑃 be the current predicate (4).
(7) Every operation takes (1,2,3), and produces new versions of (1,2,3), as well as a predicate relation 𝑅 between the old and the new model.
(8) The new predicate becomes "∃ (‘𝑡), (‘𝑠𝑡𝑎𝑐𝑘): 𝑃((‘𝑡), (‘𝑠𝑡𝑎𝑐𝑘)) ∧ 𝑅((‘𝑡), (‘𝑠𝑡𝑎𝑐𝑘), (𝑡), (𝑠𝑡𝑎𝑐𝑘))"
(9) Ask SMT to simplify the predicate.
(10) Ask SMT whether if it is satisfiable. If not, then UB has happened in the previous step.

For int2ptr (the nondeterministic step),

Example: semantics of USE-1 w.r.t. the relation in step (7)

We are using a pointer value Pointer(𝑙, 𝑡_𝑖) .

The following stands for an SMT formula that gets conjuncted into the current relation.

(1) ∃ (t ∈ ℕ) : 𝑡_𝑖 = t ∧ ‘𝑠𝑡𝑎𝑐𝑘_𝑙 contains a Unique(t) item

The "∃ t" here is a tautology; we use it to clarify that Unique contains a number not a variable.

(2) And call the topmost such item "I". Pop everything above it. Then 𝑠𝑡𝑎𝑐𝑘_𝑙 must equal the resulting value.

(3) And for all 𝑘 (𝑘 ≠ 𝑙), ‘𝑠𝑡𝑎𝑐𝑘_𝑘 = 𝑠𝑡𝑎𝑐𝑘_𝑘.

Example: semantics of int2ptr

Introduce a new tag variable 𝑡_𝑖, and conjunct into the relation that 𝑡_𝑖 is one of the permissible values. The tag variable can now safely be handled by determinism-land (1,2,3).

Exactly Mario, that's basically what I tried to formalize. The computation only has to happen once, but at any point multiple stacks may be possible for a location.

(posting this example because I have it anyway, might as well get it out of my editor)

Suppose that we have this 1 memory location, and it has a stack

𝑠𝑡𝑎𝑐𝑘₁ = [a SRW, b Unique, c SRW]

Now we do int2ptr. It returns Ptr(1, 𝑡_7) where 𝑡_7 is a fresh tag variable that's either a or c. The predicate becomes

𝑡_7 ∈ {a, c} ∧ 𝑠𝑡𝑎𝑐𝑘₁ = [a SRW, b Unique, c SRW]

Next, we do ptr2int on the pointer. If its tag was c then the stack stays the same, otherwise the last two elements are popped. The predicate becomes

(𝑡_7 = a ∧ 𝑠𝑡𝑎𝑐𝑘₁ = [a SRW])

∨ (𝑡_7 = c ∧ 𝑠𝑡𝑎𝑐𝑘₁ = [a SRW, b Unique, c SRW])

Next, suppose we push a &mut on the stack, which corresponds to pushing a Unique. The predicate becomes

(𝑡_7 = a ∧ 𝑠𝑡𝑎𝑐𝑘₁ = [a SRW, d Unique])

∨ (𝑡_7 = c ∧ 𝑠𝑡𝑎𝑐𝑘₁ = [a SRW, b Unique, c SRW, d Unique])

Now the program uses the &mut pointer for b. This creates the formula

(𝑡_7 = a ∧ 𝑠𝑡𝑎𝑐𝑘₁ = [a SRW, d Unique] ∧ false)

∨ (𝑡_7 = c ∧ 𝑠𝑡𝑎𝑐𝑘₁ = [a SRW, b Unique, c SRW, d Unique])

which SAT simplifies to

𝑡_7 = c ∧ 𝑠𝑡𝑎𝑐𝑘₁ = [a SRW, b Unique, c SRW, d Unique]

for us.
