a quick note to get it out of my head - may file an issue later: https://doc.rust-lang.org/std/fs/fn.read_dir.html

pub fn read_dir<P: AsRef<Path>>(path: P) -> Result<ReadDir>

There are two issues (as I see it) with this documentation:

  • In the source, this is qualified as io::Result, which is clearer than plain Result. But we strip that out when generating the docs. This is probably complicated to fix but I think it's worthwhile.
  • There's no (i) tag to the right indicating "Notable Traits", i.e. that this is an Iterator. I suspect that's because the iterator is inside a Result. We should probably special-case Result and Option for Notable Traits.
