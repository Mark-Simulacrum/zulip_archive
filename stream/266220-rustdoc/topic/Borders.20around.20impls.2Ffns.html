jsha (Jan 28 2022 at 02:05):

you should also be able to use the element inspector to see where the rule adding the borders is coming from. On Linux it's Ctrl-Shift-C, so probably Cmd-Shift-C on macOS. Hover around until the element with the borders appears to be highlighted, then click it- that will select the element in the Inspector tab of Dev Tools. Then look in the right-hand panel to see what border rules are active. If that panel isn't clear, you can click over to Computed, where it will show border-top-width, border-bottom-width, etc, with a little toggle to show the rule causing it and the file it came from.
