Topic: [proposal] merge tymethod and method

Noah Lev (Jan 07 2022 at 00:24):

Summary

Merge the tymethod and method URL fragments into just method.

Why?

They add needless complexity to the intra-doc links code (e.g., whether an associated item is an impl vs a prototype has to be hardcoded in some circumstances) and elsewhere in rustdoc. I don't think there's any need to distinguish between these two kinds. For example, associated constant prototypes in traits vs definitions in traits, trait impls, and inherent impls both use associatedconstant.

Drawbacks

  • It could cause slightly more fragment conflicts. Those should be solved by "semantic ID disambiguation" (#92052). I think it's unlikely that this merger would cause significantly more conflicts, though.
