Topic: wrap / overflow / scroll

jsha (Dec 30 2021 at 22:07):

I've been musing in https://github.com/rust-lang/rust/issues/92421 about what we should do when content is too big. We don't really have a coherent plan. Some places we wrap; others we allow overflow (almost always a bug); others we have a scrollbar, and others we truncate (with an ellipsis).

I propose that our default should be that everything can wrap, even if that means breaking a word (overflow-wrap: break-word). The one exception should be for code examples, which should get a horizontal scrollbar.

jsha (Jan 05 2022 at 17:22):

refining this:

  • for elements where the horizontal size is limited (i.e. all elements on our pages), default to break-word.
  • for code examples, allow horizontal scrolling.
  • for elements where the vertical size is also limited (i.e. the top bar on mobile), use ellipses
