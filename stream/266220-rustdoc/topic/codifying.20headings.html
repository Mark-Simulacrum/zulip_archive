Plain text for convenience:

In Rustdoc we use a variety of styles to separate headings from their sections:

  • Font family
  • Font size
  • Font weight
  • Indenting the section
  • Underlining the header

Some basic guidelines:

For a heading that contains identifiers, types, and/or Rust syntax (a Rusty heading), we use font-family: Source Code Pro and font-weight: 600. Most of the time these headings use font-size: 1em, though they may be bigger in a few cases.

If a Rusty heading is subordinate to another Rusty heading, we usually use indentation to distinguish the hierarchy. In that case, we use an indent of 24px.

For prose headings (generated from Markdown, or English words built into rustdoc itself), we use font-family: Fira Sans, font size, font weight: 500, and underlining. We don't use indentation. We only use underlining in the top-doc, where the prose is not subject to any outside hierarchy. Using underlining elsewhere makes the page look visually separated in a place where it's not meant to be.

When picking a font size for prose headings, be consistent with other font sizes within rustdoc: specify the font size in em, and try to use a font size already in use for headers elsewhere in rustdoc: at the time of writing: 1em, 1.1em, 1.15em, 1.3em, 1.4em, or 1.5em (with 0.8em used for subcomponents of some Rusty headers).

Avoid inverting the hierarchy: If an h3 is bold, its parent h2 should be too. A subordinate heading should not be bigger than its parents, though it may be the same size.
