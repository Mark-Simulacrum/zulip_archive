jsha (Feb 25 2022 at 06:40):

I think we're getting some good ideas in the issue thread about how to make doing it in settings have a nicer UI. @Michael notriddle Howell shared a very nice example of Pocket's UI for the equivalent task, which I like a lot. I also noticed that GMail's basic settings UI has a similar thing (though it does not offer color scheme options, presumably deferring to the OS).

I think https://github.com/rust-lang/rust/pull/93097 is a precursor to such a change. @GuillaumeGomez, want to bring that up to date with merge conflicts and the latest radio-button UI style?

Besides ease of access, one of the main themes of the thread is discoverability. For other features that we want to make sure people know about, we document them on the front page of the std docs: https://doc.rust-lang.org/nightly/std/ ([-] and source). We could do the same for themes. It's worth noting that the existence of OS settings for dark and light mode is quite well known - Android, iOS, macOS, and Windows all support it, and have put a fair amount of effort into marketing it to users. It's mainly on Linux that dark/light theme support is lagging. That's still an important group of users, of course. Many many Rust developers, myself included, are full-time Linux users. But I suspect that it won't be long before Linux (and browsers running on Linux) have support for OS setting of dark and light mode on par with other OSes.
