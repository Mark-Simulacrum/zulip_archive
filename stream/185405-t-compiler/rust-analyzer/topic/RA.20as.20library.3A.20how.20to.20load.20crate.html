Topic: RA as library: how to load crate

kangalioo (Apr 05 2022 at 07:36):

Hi! I'm trying to use rust-analyzer as a library to analyze cross-references between types in a crate to find cyclic dependencies. For that, I cobbled together this monstrosity of a function to load a crate:

fn load_crate(crate_dir: &str) -> (ra_ap_ide::AnalysisHost, ra_ap_hir::Crate) {
    let crate_dir = ra_ap_paths::AbsPath::assert(crate_dir.as_ref());

    let root = ra_ap_project_model::ProjectManifest::discover_single(crate_dir).unwrap();
    let cargo_config = ra_ap_project_model::CargoConfig {
        no_default_features: false,
        all_features: false,
        features: vec![],
        target: None,
        no_sysroot: true,
        rustc_source: None,
        unset_test_crates: ra_ap_project_model::UnsetTestCrates::All,
        wrap_rustc_in_build_scripts: true,
    };
    let workspace =
        ra_ap_project_model::ProjectWorkspace::load(root, &cargo_config, &|_progress| {}).unwrap();

    let (host, vfs, _) = ra_ap_rust_analyzer::cli::load_cargo::load_workspace(
        workspace,
        &ra_ap_rust_analyzer::cli::load_cargo::LoadCargoConfig {
            load_out_dirs_from_check: true,
            with_proc_macro: false,
            prefill_caches: false,
        },
    )
    .unwrap();
    let db = host.raw_database();

    let crate_ = ra_ap_hir::Crate::all(db)
        .into_iter()
        .find(|crate_| {
            let vfs_path = vfs.file_path(crate_.root_file(db));
            // go from `project/src/lib.rs` to `parent`
            let root = vfs_path
                .as_path()
                .unwrap()
                .parent()
                .unwrap()
                .parent()
                .unwrap();
            root == crate_dir
        })
        .unwrap();

    (host, crate_)
}

This works:

let (host, crate_) = load_crate("/home/kangalioo/dev/rust/poise");
let db = host.raw_database();
println!("found crate {:?}", crate_.display_name(db));

Is my implementation of load_crate the best way to load a single crate? Or is there something shorter, that ideally also doesn't require depending on five different ra_ap_* crates?

Florian Diebold (Apr 05 2022 at 08:24):

that seems pretty normal. the APIs aren't yet optimized for ease of use (or really for use outside of RA itself at all), and there are lots of complexities. Only thing I'd question is whether this is the best way to find the Crate

Florian Diebold (Apr 05 2022 at 08:27):

you could use load_workspace_at to make it a bit shorter

Florian Diebold (Apr 05 2022 at 08:33):

for finding the crate I'd at least do vfs_path.as_path().map_or(false, |p| p.starts_with(crate_dir)) instead of all the unwrapping and directly comparing paths
