matklad (Oct 14 2020 at 13:42):

And I have a feeling that our "good first issues" are quite some rabbit holes.

That's intentional. Criteria for "good first issue" is "no need to understand the broad context". The issue itself can be huge, as long as it can be isolated from the rest of the compiler.

For example, "implement formatter" would be a good first issue (relatively narrow API, syntax tree in, syntax tree out), "implement auto-import" would not (broad api, syntax, semantics, assists, search, etc)
