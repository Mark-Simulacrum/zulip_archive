Hey, today I've found myself debugging "things are slow because diagnostics are computed eagarly" issues for the third time, so I introduced a new label: Broken Window. It is for stuff which is not naturally high priority, but nontheless should be fixed as fast as possible, to avoid accumulation of technical debt and minor brokenness.

Roughly, any panic we see is a broken window.
