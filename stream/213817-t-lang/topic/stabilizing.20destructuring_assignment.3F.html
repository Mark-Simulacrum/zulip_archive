Daniel Henry-Mantilla (Apr 20 2021 at 11:09):

I personally believe that the generalization to "branched" assignments would be to feature in patterns.
Granted, it would allow writing let in x = …; for the same semantics as x = …;, but that's already the case with ref (let ref x = …; vs. let x = &…;), and it would require a keyword, albeit a very intuitive one.
It would, however, avoid having to write that ugly Pat(binding_) => { binding = binding_; … }.
