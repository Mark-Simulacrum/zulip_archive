Hello, I realize this is probably largely a non-goal, but I kinda wanted to verify,
There was a proposal for variable-length arrays a while back, something like[T; dyn var].
There is some current work on unsized locals.
However, obviously that would overlap somewhat with const generic sizing of arrays and array-like things.
From your perspective, would [T; dyn var] be the same type or different from [T; const N]?
I am asking because I am contemplating The Future and am trying to figure out if we will have both Simd and DynSimd as types, just Simd<T, const N>, or Simd<T, {const,dyn} N>, or if I can calibrate expectations in any other direction.

bjorn3 (Dec 19 2021 at 22:09):

It will have to be a different type. [T; N] has a fixed size and implements Sized. [T; dyn var] doesn't have a fixed size and thus doesn't implement Sized. [T; dyn var] would be more similar to [T].
