@Stefan dns2utf8 Schindler I believe the difference is that #project-const-generics is focused on stabilising the fn foo<T: const usize> feature and #t-compiler/const-eval is focused on improving const fn support.

So both const related but one is focused on using them in generics and static dispatch and the other for running general purpose rust code.
