I definitely think we should revisit the privacy and data access rules this year. As @Nick Cameron noted, the "summary blog posts" don't contain much actionable data. Further, what they do have is encoded in png format, and not even available in CSV.

nikomatsakis (Jul 06 2021 at 15:15):

This is probably best coordinated with Foundation folks, though.

Nick Cameron (Jul 06 2021 at 22:35):

Yeah, offering complete aggregate stats seems like it would be useful and privacy-sensitive. We'd probably want some small floor number as a minimum for any category (e.g., if there are only two people using Rust for some obscure platform, then they should be forced into 'other'). Even more so for any correlation we make available. If we offer to make cross-correlations available on request it could be a lot of work, so we should probably limit the timeframe that offer is open and only make it open to teams.
