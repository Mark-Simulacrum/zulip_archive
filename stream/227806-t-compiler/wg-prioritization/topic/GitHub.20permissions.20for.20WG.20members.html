Léo Lanteri Thauvin (Apr 14 2020 at 15:29):

I was wondering if members of the WG-prioritization should have triage permissions on the rust-lang/rust repo. According to the GitHub docs, it grants the following permissions:

  • Apply labels (Useful for P-*, I-prioritize, regressions...)
  • Close, reopen, and assign all issues and pull requests (Useful for assigning people to work on issues when it is deemed necessary)
  • Apply milestones (Maybe usefull for defining release blockers?)
  • Mark duplicate issues and pull requests (Would probably be done by T-release folks)
  • Request pull request reviews (pretty much useless because we have r?

So to me the question is: should we always let leaders assign those labels, which would make sense as they "represent" the WG and are responsible for the final decisions? Or should we give those permissions to all members so everyone can do it in case leaders are AFK/unavailible, or maybe just because is isn't fun for one person to always do the work.
